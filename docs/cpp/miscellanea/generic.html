Generic utilities¶

A number of utilities to compute quantities that are of general relevance.

double pagmo::uniform_real_from_range(double lb, double ub, detail::random_engine_type &r_engine)¶

Generates a random number within some lower and upper bounds.

Creates a random number within a closed range. If both the lower and upper bounds are finite numbers, then the generated value \( x \) will be such that \(lb \le x < ub\). If \(lb == ub\) then \(lb\) is returned.

NOTE: This has to be preferred to std::uniform_real<double>(r_engine) as it performs checks that avoid undefined behaviour in pagmo.

Example:

std::mt19937 r_engine(32u);
auto x = uniform_real_from_range(3,5,r_engine); // a random value
auto x = uniform_real_from_range(2,2,r_engine); // the value 2.

Return
a random floating-point value
Parameters
  • lb: lower bound
  • ub: upper bound
  • r_engine: a std::mt19937 random engine
Exceptions
  • std::invalid_argument: if:
    • the bounds contain NaNs or infs, or \( lb > ub \),
    • if \(ub-lb\) is larger than implementation-defined value

vector_double pagmo::random_decision_vector(const std::pair<vector_double, vector_double> &bounds, detail::random_engine_type &r_engine)¶

Generates a random decision vector.

Creates a random decision vector within some bounds. If both the lower and upper bounds are finite numbers, then the \(i\)-th component of the randomly generated pagmo::vector_double will be such that \(lb_i \le x_i < ub_i\). If \(lb_i == ub_i\) then \(lb_i\) is returned.

Example:

std::mt19937 r_engine(32u);
auto x = random_decision_vector({{1,3},{3,5}}, r_engine); // a random vector
auto x = random_decision_vector({{1,3},{1,3}}, r_engine); // the vector {1,3}

Return
a pagmo::vector_double containing a random decision vector
Parameters
  • bounds: an std::pair containing the bounds
  • r_engine: a std::mt19937 random engine
Exceptions
  • std::invalid_argument: if:
    • the bounds are not of equal length, they have zero size, they contain NaNs or infs, or \( \mathbf{ub} < \mathbf {lb}\),
    • if \(ub_i-lb_i\) is larger than implementation-defined value

std::vector<std::vector<vector_double::size_type>> pagmo::kNN(const std::vector<vector_double> &points, std::vector<vector_double>::size_type k)¶

K-Nearest Neighbours.

Computes the indexes of the k nearest neighbours (euclidean distance) to each of the input points. The algorithm complexity (naive implementation) is \( O(MN^2)\) where \(N\) is the number of points and \(M\) their dimensionality

Example: 

auto res = kNN({{1, 1}, {2, 2}, {3.1, 3.1}, {5, 5}}, 2u);

Return
An std::vector<std::vector<population::size_type> > containing the indexes of the k nearest neighbours sorted by distance
Parameters
  • points: the \(N\) points having dimension \(M\)
  • k: number of neighbours to detect
Exceptions
  • std::invalid_argument: If the points do not all have the same dimension.
